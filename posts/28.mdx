## What's Sajinpage?

사진페이지는 제가 갑작스럽게 생각이 난 아이디어에서 출발한 아이디어입니다.
무슨 아이디어냐 하면, 이미지는 웹사이트에서 2가지 형태로 출력됩니다. 하나는 이미지 파일로서 출력되며, 또 하나는 파일이 아닌 코드의 형태로 출력됩니다.이러한 것을 Base 64 이미지라고 부릅니다.

제가 생각하기로 Base 64 이미지는 코드로 표현되니 url의 Parameter로 받을 수 있다고 생각했습니다. 그러하여 제가 생각한 아이디어는 이미지를 받아서, 그 이미지를 Base 64의 형태로 바꾼후 `https://localhost:3000/viewer/(base64코드)` 형태로 사용자가 올린 이미지를 url로만 접근하여 볼 수 있게 하는 것이었습니다.

그치만, 제가 여기서 한 가지 간과한 사실이 있었습니다. 바로 Base 64 텍스트 크기가 엄청나게 크다는 것이었는데요. 이것은 바로 사용자가 그 URL 형태의 파일을 어딘가에 저장할 수 없다는 결과로 도출되었습니다.

그러하여 기존의 아이디어를 완전히 접은체 Sajin을 개발을 진행했으며, 수정된 아이디어는 파일을 DB에 올리고, UID를 무작위로 생성후 `viewer/(uid)` 로 접근시 사용자가 올린 이미지를 볼 수 있게 하였습니다.

원래는 Sajin이라는 이름으로 2022년경에 약 1달 동안 개발후 서비스를 개발하였지만, 로직 부분과 UI & UX 부분이 너무 빈약하여 서비스를 끝내 출시하지 못했습니다.

그렇지만 이번에 완전히 새로 구축한 Sajinpage도 로직적인 측면과 UI & UX 부분이 데스크탑 부분에서는 마음에 들지는 않지만, 그래도 성공적으로 프로젝트를 완성하여 회고로 남겨볼까 합니다.

## 사진페이지의 기술 스택

저는 아직까지는 Next.js 도입이 저의 프로젝트에서 조금은 과하다고 판단되어 Vite + Yarn + TypeScript으로 개발 환경을 구축한 후에, React + React Router + Jotai를 사용하여 앱의 전반적인 로직을 구축하였고, 서버는 Firebase가 아닌 Supabase를 이번에 새로 사용하여 구축하였습니다.

또한 도메인 구입은 Google Domains로 구입하였으며, 배포는 Vercel에서 배포하였습니다.
