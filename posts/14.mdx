주로 처음 웹 개발에 입문한다면 하는 과정이 있다. 프론트엔드를 할 것이냐. 백엔드를 할 것이냐 이다.

간단하게 예기하자면, 시각적인 것을 좋아하고, 웹 앱 개발이나 앱 개발을 하고 싶다면 프론트엔드이며, 백엔드는 시각적인 것을 좋아하면서 서버를 구현하고 싶고, API 구현 등이 즐겁다면 백엔드 개발자가 되면 된다.

나는 프론트엔드를 좋아하면서 백엔드는 Supabase를 사용하며 즐겁게 백엔드를 쉽고 값 싸게 사용하면서 프론트엔드는 안정되고 값비싸게 만들고 있다.

언제나 진심으로 프론트엔드를 구현하며 드는 생각은 아무리 열심히 떨어지지 않고 안정되게 걸기 위해 그림 액자를 힘들게 만들더라도 그림이 부실하면 그 그림과 그림 액자는 의미가 없어지는 것이라는 생각이 든다.

아무리 힘들고 안정되게 로직을 구현하더라도 외부에서 CSS 코드나 GSAP 코드 잘못쓰게 된다면 한 순간에 사용자는 오류를 보게 된다면 즉시 서비스를 떠나게 된다.

즉 아무리 Frame을 잘 짜도 마지막은 그림인 것이다.
프론트엔드 개발자에게 말하고 싶다. 아무리 로직을 힘들게 짰다고 해도 UI UX가 중요하다고.
