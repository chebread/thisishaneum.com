## 현재의 웹 셍태계

현재는 나도 그렇고, 다른 개발자들도 그렇듯이 Node.js 기반의 프레임워크, 라이브러리를 사용하고 있다.
React만을 사용한다면, SPA의 방식을 사용한다면 이러한 방식으로 개발할 것이다.

- Node.js + CRA + React + RRD + Css in JS + ...

그리고, Next.js를 사용한다면 이러한 방식으로 개발할 것이다.

- Node.js + Next.js + React + Css in JS + ...

## 미래의 웹 생태계

그치만, 이제 Node.js의 시대가 저물고 있음을 현재의 출시되는 것들에 대해서 추측하며, 확신할 수 있을 것이다.

- Pnpm
- Bun
- Vite
- Deno
- ...

이러한 것들의 특징은 "속도" 이다. 그런데 위에서 모두들 Node.js를 사용하지만, Deno는 Node.js를 사용하지 않는다, 그것은 Deno 그 자체이다.
